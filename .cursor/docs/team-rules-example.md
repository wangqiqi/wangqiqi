# Team Rules Example - Cursor AI协作规范

*此文件是团队规则配置的示例参考。根据Cursor官方文档，团队规则应通过Cursor控制台集中管理。*

## 🤝 团队协作规范

### 代码审查标准
- 所有PR需要至少一人审查
- 重点检查安全性和性能问题
- 鼓励建设性反馈和改进建议

### 沟通原则
- 使用清晰、专业的语言
- 及时响应团队成员的询问
- 主动分享重要信息和更新

### 项目管理
- 遵循敏捷开发流程
- 按时参加团队会议
- 及时更新任务状态

## 🛡️ 安全与合规

### 数据保护
- 严格遵守数据保护法规
- 不分享敏感信息
- 使用加密传输重要数据

### 访问控制
- 遵循最小权限原则
- 定期审查和更新权限
- 及时吊销离职员工权限

## 📋 开发规范

### 代码质量
- 遵循团队编码标准
- 编写完整的单元测试
- 确保代码的可维护性

### 文档要求
- 为复杂功能编写技术文档
- 更新API文档
- 维护项目README

## 🚀 部署与运维

### 发布流程
- 遵循CI/CD流程
- 执行自动化测试
- 进行生产环境验证

### 监控告警
- 设置关键指标监控
- 配置合理的告警阈值
- 及时响应系统告警

---

## Cursor控制台配置说明

1. 访问 [Cursor控制台](https://cursor.com/console)
2. 导航到 Team Rules 部分
3. 创建新规则并粘贴上述内容
4. 设置强制执行选项（如果需要）
5. 激活规则使其生效

## 注意事项

- 团队规则对所有团队成员强制执行（除非特别设置）
- 规则冲突时：Team Rules → Project Rules → User Rules
- 定期审查和更新团队规则以适应项目发展
